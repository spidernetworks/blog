Understanding IP Addressing: A Complete Guide

By Sarah Networker

Network Engineer at WebTech

Introduction to IP Addressing

IP (Internet Protocol) addressing is the backbone of modern networking. Every device connected to a network—be it a local network or the global internet—requires a unique IP address to communicate with other devices.

IPv4 vs IPv6

There are two versions of IP addresses in use today:

IPv4

  • 32-bit address (4 bytes)
  • Dotted decimal notation (e.g., 192.168.1.1)
  • Approximately 4.3 billion unique addresses
  • Most widely used currently

IPv6

  • 128-bit address (16 bytes)
  • Hexadecimal notation (e.g., 2001:0db8:85a3::8a2e:0370:7334)
  • Approximately 3.4×10³⁸ unique addresses
  • Designed to replace IPv4

IPv4 Address Structure

An IPv4 address consists of 32 bits divided into four octets (8 bits each), represented in dotted decimal notation:

192   .168   .1   .1
11000000.10101000.00000001.00000001 (Binary)

Note: Each octet can range from 0 to 255 in decimal (00000000 to 11111111 in binary).

IP Address Classes

IPv4 addresses are divided into five classes (A, B, C, D, E) based on the first few bits:

Class Range Network Bits Host Bits Use
A 1.0.0.0 to 126.255.255.255 8 24 Large networks
B 128.0.0.0 to 191.255.255.255 16 16 Medium networks
C 192.0.0.0 to 223.255.255.255 24 8 Small networks
D 224.0.0.0 to 239.255.255.255 - - Multicasting
E 240.0.0.0 to 255.255.255.255 - - Experimental

Subnet Mask

A subnet mask determines which portion of an IP address is the network portion and which is the host portion. Common subnet masks include:

  • Class A: 255.0.0.0 (/8)
  • Class B: 255.255.0.0 (/16)
  • Class C: 255.255.255.0 (/24)

Practical Examples

Let's examine a common home network configuration:

IP Address: 192.168.1.100
Subnet Mask: 255.255.255.0 (or /24)
Network Address: 192.168.1.0
Broadcast Address: 192.168.1.255
Usable Host Range: 192.168.1.1 to 192.168.1.254

Pro Tip: The first address in a subnet is the network address, and the last is the broadcast address. These cannot be assigned to hosts.

Conclusion

Understanding IP addressing is fundamental for anyone working with networks. Whether you're configuring a home router or designing enterprise networks, these concepts form the foundation of all network communication. As IPv6 adoption grows, the need for NAT (Network Address Translation) will diminish, but the underlying principles of addressing and routing remain essential knowledge.
